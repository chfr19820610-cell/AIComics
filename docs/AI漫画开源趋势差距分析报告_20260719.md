## 三、AI-Comic-Generator (Dapeng960208) 深度分析

**Stars**: ~200+ | **技术栈**: FastAPI + Vue3 + Google Gemini | **协议**: MIT

### 核心特色功能
1. **角色工坊 (Character Workshop)** — 独立的角色设计与设定图生成模块，包含人物形象可视化
2. **一致性检查服务** (`consistency_service.py`) — 专门的一致性检查 API，确保跨分镜角色外貌统一
3. **故事板可视化编辑器** — Vue3 前端提供直观的分镜编辑界面
4. **Gemini Pro Image 原生图像生成** — 直接利用 Gemini 的多模态能力，无需额外部署 ComfyUI
5. **批量绘图** — 支持批量生成分镜图片
6. **全 Web 操作** — 纯浏览器端操作，无需安装本地模型

### 差距项 (AIComics 当前缺失)
- 角色工坊与一致性检查已落地运行（AIComics 仅路线图中）
- 纯 API 调用无需本地 GPU（AIComics 强依赖 ComfyUI + SDXL）

## 四、AIComicBuilder (LingyiChen-AI) 深度分析

**Stars**: ~500+ | **技术栈**: Next.js 16 + React 19 + SQLite + Drizzle ORM | **协议**: Apache 2.0

### 核心特色功能
1. **多模型供应商** — 同时支持 OpenAI / Gemini / Imagen / Kling / Seedance / Veo
2. **角色四视图生成** — 从剧本自动提取角色并生成正面/侧面/背面/3/4视图
3. **参考图模式重构 (v0.2.2)** — 角色一致性通过参考图实现
4. **分镜看板 (Kanban)** — 按生成进度自动分列，支持版本管理
5. **AI 视频生成集成** — Seedance / Kling / Veo 三大视频模型
6. **国际化 (next-intl)** — 多语言界面支持
7. **提示词管理** — 可编辑和管理所有 AI 提示词模板
8. **分镜 AI 优化** — AI 辅助优化分镜质量

### 差距项 (AIComics 当前缺失)
- 多模型供应商切换（AIComics 仅 ComfyUI/SDXL 一条路）
- 角色四视图生成
- 分镜版本管理与看板视图
- AI 视频生成 (Seedance/Kling/Veo) — AIComics 仅 FFmpeg 图片+配音组合
- Docker 部署支持

## 五、Toonflow (HBAI-Ltd) 深度分析

**Stars**: **11.6k** | **技术栈**: TypeScript + Express 5 + Electron 40 + SQLite | **协议**: Apache 2.0

### 核心特色功能
1. **无限画布生产工作台** — 类无限画布组织剧本、角色、分镜、素材与视频节点
2. **三层 Agent 协作体系** — 决策层、执行层、监督层协同工作
3. **Agent 持久化记忆** — 基于 ONNX 向量检索的跨会话记忆系统
4. **可编程供应商系统** — 直接在 UI 中编写供应商 TypeScript 逻辑，无需改源码
5. **章节事件图谱** — 自动提取原著章节事件并结构化存储
6. **Skill 文件化配置** — ScriptAgent/ProductionAgent 核心提示词外化为 Markdown Skill
7. **7 种语言界面** — 中/繁/英/泰/越/日/俄
8. **跨平台桌面应用 (Electron)** — Windows/Linux/macOS 原生安装包
9. **Docker + 云平台部署** — 智星云镜像一键部署
10. **完善的成本核算** — Demo 示例给出明确的各模型费用明细 ($130/2分钟成片)

### 差距项 (AIComics 当前缺失) — 最重要的参考
- **Agent 体系**：Toonflow 的三层 Agent + 持久化记忆是架构级优势，AIComics 仍为线性流水线
- **无限画布**：超越线性的创作组织方式
- **可编程供应商**：用户可自行编写供应商插件，AIComics 仅内置 ComfyUI/Piper
- **事件图谱**：长文本处理的信息结构化方法
- **多语言**：7 语言界面
- **Electron 桌面端**：AIComics 仅 Web SPA
- **社区规模**：11.6k Stars vs AIComics 未开源或未公开

## 九、2026 年 AI 漫画开源趋势总结

1. **角色一致性已成标配**：所有活跃项目 (AI-Comic-Generator、AIComicBuilder、Toonflow、AIMangaStudio) 均将角色管理作为核心能力。AIComics 的 LoRA 路线图需要加速。

2. **从"图片拼接"到"AI 视频生成"**：2026 年趋势是直接集成 Seedance / Kling / Veo 等视频生成模型（AIComicBuilder v0.2.2 重构参考图模式、Toonflow 默认 Seedance 2.0），代替传统的图片+FFmpeg 合成。AIComics 当前仅用 FFmpeg 拼接，技术路线需要评估是否需要升级。

3. **Agent 化协作架构**：Toonflow 的 11.6k Stars 验证了三层 Agent + 持久化记忆的架构方向。单一线性流水线正在被多 Agent 协作体系取代。

4. **多模型供应商 > 单一模型绑定**：AIComicBuilder 和 Toonflow 都支持用户自由切换多家模型，降低供应商锁定风险。

5. **桌面端 + Docker 双轨部署**：Electron 桌面端提供原生体验，Docker 提供云端部署灵活性。

6. **开源社区运营至关重要**：Toonflow (11.6k)、AIMangaStudio (1.4k)、AIComicBuilder (~500+) 的 Star 数与活跃度差异证明社区运营对项目成功的影响。
