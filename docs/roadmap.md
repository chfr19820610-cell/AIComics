## 🔴 P0 — 立即投入（竞品分析建议）

以下 6 项 P0 建议来自竞品差距分析，是当前行业标配功能，缺失即构成明显短板。

### P0-1: AI 编剧引擎 — 复用供应商抽象层

| 维度 | 内容 |
|------|------|
| **问题** | 当前故事生成硬编码在管线中，无独立编剧引擎模块，无法灵活切换 LLM 供应商或自定义编剧策略 |
| **参考竞品** | **AIComicBuilder**（AI SDK 剧本生成）+ **Toonflow**（三层 Agent 驱动剧本改编，ScriptAgent + Skill 文件化）+ **AI-Comic-Generator**（Gemini 故事→分镜管线） |
| **用户价值** | 🔥🔥🔥 用户可自由选择编剧 LLM、自定义编剧风格、复用已有剧情模板 |
| **实现方案** | ① 定义 `ScreenplayEngine` 抽象接口（继承 Provider 抽象层） ② 实现 LLM Provider 路由——根据配置自动选择 OpenAI / Gemini / Claude / 本地模型 ③ 支持剧本结构模板（三幕/四幕/漫画分镜格式） ④ 输出标准化 `ScreenplaySchema`（角色/场景/对白/镜头提示） ⑤ 保留当前剧本→分镜管线作为默认实现 |
| **可行性评估** | ✅ **高** — 复用 P0-1 Provider 抽象层接口设计；现有 LLM 故事生成逻辑可封装为默认 Engine 实现 |
| **工期估算** | **~4-6 天** |
| **前置依赖** | Provider 抽象层（已立项） |

### P0-2: ComfyUI 面板插件 — 占领 ComfyUI 生态

| 维度 | 内容 |
|------|------|
| **问题** | AIComics 深度依赖 ComfyUI，但在 ComfyUI 社区中无可见性，缺乏 AIComics 专属的自定义节点 |
| **参考竞品** | **comfyui_panels**（bmad4ever，漫画面板布局节点）+ **CR Comic Panel Templates**（Comfyroll Studio，预设面板模板）+ **Consistent Character Creator 3.0**（Qwen Image Edit 角色一致性工作流） |
| **用户价值** | 🔥🔥🔥 占领 ComfyUI 漫画/漫剧生态位；吸引 ComfyUI 用户群；反哺 AIComics 产品知名度 |
| **实现方案** | ① 开发 `ComfyUI-AIComics-Nodes` 自定义节点包：面板布局节点、角色参考图节点、分镜→视频拼接节点 ② 发布至 ComfyUI Manager 官方注册列表 ③ 提供预设 Workflow JSON 模板（漫画生成/漫剧合成/角色一致） ④ 编写英文文档 + 效果展示图 ⑤ GitHub 单独仓库运营，与 AIComics 主仓库互链 |
| **可行性评估** | ✅ **高** — AIComics 已有完整的 ComfyUI 集成经验（workflow JSON、Docker sidecar），插件开发无技术障碍 |
| **工期估算** | **~5-8 天** |
| **前置依赖** | 无（可独立推进） |

### P0-3: 多视频引擎接入 — Wan2.1 / CogVideo / Veo

| 维度 | 内容 |
|------|------|
| **问题** | 当前视频管线仅 AnimateDiff（SD1.5，低分辨率 256×144）+ FFmpeg 拼接；Seedance 集成方案已立项但尚未落地，且仅单一引擎存在供应商锁定风险 |
| **参考竞品** | **AIComicBuilder** 支持 Seedance / Kling / Veo 三大视频引擎 + 参考图模式 v0.2.2；**Toonflow** 默认 Seedance 2.0 但计划接入更多；**Wan2.2** 已成为 ComfyUI 原生支持的本地视频方案 |
| **用户价值** | 🔥🔥🔥 从"幻灯片"到真正 AI 视频；本地（Wan2.2）+ 云端（Veo/CogVideo）双轨覆盖；多引擎自动回退 |
| **实现方案** | ① **Wan2.2 TI2V-5B**（本地优先）— 按 `wan22_migration_plan.md` 执行：更新 ComfyUI → 下载 5B 模型 → 创建 Wan2.2 workflow JSON ② **CogVideo**（云端备选）— 新增 `CogVideoProvider` 实现 video 接口 ③ **Veo 3.1**（云端高画质）— 新增 `VeoProvider`，支持叙事控制 + 原生音频 ④ 引擎选择策略：本地模式 Wan2.2 → 云端模式 Veo → 降级 FFmpeg 拼接 |
| **可行性评估** | ⚠️ **中高** — Wan2.2 方案已在迁移计划中（`wan22_migration_plan.md`），本地模型需下载约 5GB；CogVideo/Veo 需 API 对接和 Key 配置 |
| **工期估算** | **Wan2.2: ~2-3 天** / CogVideo: ~2-3 天 / Veo: ~2-3 天（总计 ~5-8 天，可分阶段交付） |
| **前置依赖** | Provider 抽象层（已立项）；Wan2.2 需 ComfyUI 更新 + 模型下载 |

### P0-4: 智能面板排版 — 从逐帧到整页

| 维度 | 内容 |
|------|------|
| **问题** | 当前输出是逐帧图片+视频，无漫画风格的页面排版能力——无法将生成的分镜图排列为标准的漫画/漫画面板布局 |
| **参考竞品** | **AIMangaStudio**（1.4k⭐）AI 分镜自动排版（对话框、镜头切换）+ **AI Comic Factory**（1.3k⭐）面板生成引擎 + **ComfyUI Manga Workflows**（comfyui_panels + CR Comic Panel Templates 预设面板布局） |
| **用户价值** | 🔥🔥🔥 从"图片序列"到"可读的漫画页面"；提升成品观感；同时满足静态漫画和动态漫剧输出 |
| **实现方案** | ① 定义 `PageLayout` 数据模型（面板数 / 排列方式 / 尺寸比例 / 页面方向） ② 实现面板排版引擎：支持常见布局（4格/6格/自由/不规则）、根据分镜重要性自动分配面板大小 ③ ComfyUI 节点包装：`AIComics Page Layout` 节点，输出布局后的整页图片 ④ 导出集成：PNG 单页 / PDF 多页 / 视频逐帧传递布局信息 |
| **可行性评估** | ✅ **中高** — 布局算法复杂度可控；ComfyUI 已有面板节点可作为参考实现；AIComics 已有的图片管线可直接输出单页合成结果 |
| **工期估算** | **~5-7 天** |
| **前置依赖** | 无（可独立推进，推荐与 P0-2 ComfyUI 插件联动开发） |

### P0-5: Web Demo 加强 — 获客基础设施

| 维度 | 内容 |
|------|------|
| **问题** | AIComics 已有 Web SPA（FastAPI + React + JWT/OIDC 登录），但当前作为开发工具而非获客产品——缺乏引导式体验、Demo 模式和公开访问能力 |
| **参考竞品** | **Toonflow**（智星云镜像一键部署 + 可在线体验 Demo）+ **AIComicBuilder**（Next.js 完整 Web 应用，可直接体验）+ **AI-Comic-Generator**（Vue3 全 Web 操作，无需安装） |
| **用户价值** | 🔥🔥🔥 降低新用户体验门槛；将 AIComics 从"CLI 工具"升级为"可演示的产品"；支撑开源获客 |
| **实现方案** | ① **引导式创作流**：输入故事梗概 → 逐步配置角色/风格 → 一键生成预览 → 导出/分享 ② **游客 Demo 模式**：无需登录即可体验（限制产出数量 + 加水印） ③ **云 Demo 实例**：Docker 部署 + 公网域名 + 资源限制（单用户限时 5 分钟/单次生成 1 集） ④ **生成进度展示**：WebSocket 实时推送管线各阶段状态（剧本→分镜→图片→配音→视频） ⑤ **效果画廊**：展示 AIComics 生成的最佳作品 + 视频预览 |
| **可行性评估** | ✅ **高** — Web 前端已有（React SPA），后端 API 已有（FastAPI），增强为主而非新建 |
| **工期估算** | **~5-7 天**（核心增强）+ **~2 天**（云部署 + 域名） |
| **前置依赖** | Docker 部署（已立项）；建议先完成 P0-1 Provider 抽象层以确保 Demo 体验稳定 |

### P0-6: 对白气泡 — 成品完整体验

| 维度 | 内容 |
|------|------|
| **问题** | AIComics 当前输出视频的字幕为 ASS 格式压制在画面底部，无漫画风格的对白气泡（speech bubble/balloon），成品缺乏"漫画感" |
| **参考竞品** | **AIMangaStudio**（对话框渲染，自动气泡排版）+ **AI Comic Factory**（面板内气泡生成）+ **comic-generator**（对话气泡定位）+ **ComfyUI CR Comic Panel Templates**（含文本框节点） |
| **用户价值** | 🔥🔥 从"带字幕的视频"到"真正的漫画成品"；提升分享意愿和品牌识别度 |
| **实现方案** | ① **图片模式**（静态漫画）：在面板排版阶段（P0-4）将对白渲染为漫画气泡——支持标准气泡形状（椭圆/圆角矩形/云朵/锯齿）、角色指示尾尖（tail/tailess）、多行文本自动换行 ② **视频模式**（动态漫剧）：在每帧上叠加动态气泡——气泡根据台词时间出现/消失（fade-in/fade-out）、支持打字机效果（文字逐字出现） ③ **气泡位置引擎**：根据画面构图自动计算气泡锚点（避开角色面部/主体）；支持手动微调（前端 UI） ④ **样式系统**：多套气泡主题（经典漫画/极简/手绘风/日漫风）+ 字体配置 |
| **可行性评估** | ⚠️ **中** — 图片模式实现较简单（PIL/cairo 渲染），视频模式需逐帧处理（性能开销大）；气泡位置算法需要调优 |
| **工期估算** | **图片模式: ~2-3 天** / 视频模式: **~3-5 天**（总计 ~5-7 天，图片模式可先交付 P0） |
| **前置依赖** | P0-4 智能面板排版（气泡定位依赖页面布局信息） |

## 工程风险与注意事项

1. **ComfyUI 插件维护成本**：发布后需持续维护版本兼容性（ComfyUI API 变更频繁），建议设计为薄封装层，核心逻辑仍在 AIComics 主仓库
2. **视频引擎模型体积**：Wan2.2 5B 模型约 5GB，14B 约 24GB——需在文档中明确最低硬件要求（推荐 8GB+ VRAM）
3. **Web Demo 防滥用**：游客模式需限流（IP + Cookie 限频）、输出水印、限制单次生成长度，防止资源耗尽
4. **对白气泡性能**：视频模式逐帧渲染气泡在 1080p 下可能成为性能瓶颈，建议预渲染+缓存策略
5. **竞品跟进**：Toonflow 11.6k⭐ 持续迭代，AIComicBuilder v0.2.6 已集成三大视频引擎——AIComics 需差异化定位（深度本地化、全自有发布管线、ComfyUI 生态优势）
