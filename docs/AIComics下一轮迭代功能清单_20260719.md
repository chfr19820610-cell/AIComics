### 🔴 P0 — 立即投入（2-3 周）

#### P0-1: 多模型供应商抽象层

| 维度 | 内容 |
|------|------|
| **问题** | AIComics 当前仅绑定 ComfyUI + SDXL 一条图片生成链路，无回退机制 |
| **参考** | AIComicBuilder 支持 OpenAI/Gemini/Imagen/Kling/Seedance/Veo |

| **实现方案** | ① 定义 `Provider` 抽象基类（text/image/video 三类接口） ② 实现 ComfyUI Provider（现有）、OpenAI Provider（DALL-E）、Gemini Provider（Imagen） ③ 前端设置页 → 提供配置 UI ④ 运行时根据配置自动路由 |
| **工程量** | ~3-5 天（核心接口设计 + 2 个 Provider 实现 + 前端配置页） |

#### P0-2: AI 视频生成集成（Seedance 2.0 优先）

| 维度 | 内容 |
|------|------|
| **问题** | 当前仅 FFmpeg 做静图+配音拼接，视频质量远不及竞品 |

## 五、关键 API / 集成参考

### Seedance 2.0 API
- **Endpoint**: ByteDance ModelArk / Dreamina API
- **Capabilities**: Image-to-Video, Text-to-Video, Multi-modal (image+video+audio+text)
- **Character Consistency**: 支持最多 9 张参考图 → 角色保持一致
- **Resolution**: 1080p (2.0), 4K (2.5)
- **Max Duration**: 15s (2.0), 30s (2.5)
- **Audio**: 原生音频同步

### Kling 3.0 API
- **Endpoint**: kling.ai/dev
- **SDK**: Python / Node.js / Java
- **Capabilities**: Image-to-Video, Text-to-Video, Motion Control, Multi-shot
- **Specialty**: 动漫/漫画风格生成质量高于写实
- **Lip Sync**: 支持

### Veo 3 / 3.1 API
- **Access**: via Google AI Studio / Leonardo.ai / API
- **Capabilities**: Text-to-Video, Image-to-Video, Native Audio
- **Max Duration**: 8s (3.0), 8s+ (3.1)
- **Resolution**: 720p-1080p

### ComfyUI Manga/Comic Workflows
- **comfyui_panels** (bmad4ever): 漫画面板布局节点
- **CR Comic Panel Templates** (Comfyroll Studio): 预设漫画面板模板
- **Consistent Character Creator 3.0**: Qwen Image Edit 模型驱动的角色一致性工作流
- **ToonCrafter for ComfyUI**: 卡通插值工作流

## 七、风险与注意事项

1. **API 成本**：Seedance/Kling/Veo 均为商业 API，接入后 AIComics 从"全本地免费"变为"依赖云 API + 按量付费"。需保留 ComfyUI 本地模式作为免费回退。
2. **角色一致性质量**：即使是 Seedance/Kling 也无法保证 100% 角色一致。参考图模式 + 人工审核不可替代。
3. **架构演进**：P0 阶段保持单体架构不变；P2 的 Agent 化改造需提前规划接口契约。
4. **开源竞争**：Toonflow 已 11.6k⭐ 且持续活跃，AIComics 需突出差异化定位（如深度 ComfyUI 集成、全本地运行、自由发布流程）。
