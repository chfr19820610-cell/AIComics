## 2. Hacker News — Show HN

### Title (≤80 chars)
```
Show HN: AIComics – Open-source AI animation pipeline, zero API cost, local-only
```

### Body
```
I built AIComics — a fully local AI animation pipeline that turns story ideas into animated comic episodes. Everything runs on your own machine: no API calls, no subscriptions, no cloud dependency.

Pipeline:
1. You provide a story outline
2. LLM generates a shooting script with shot breakdowns
3. ComfyUI + SDXL renders anime-style keyframes for each shot
4. Piper TTS generates Mandarin voiceover for each shot
5. FFmpeg assembles frames + audio into video episodes
6. Optional auto-publish to Bilibili/Xiaohongshu/Douyin via social-auto-upload

Why I built it:
Most "AI video" tools are wrappers around paid APIs or cloud services. I wanted a pipeline where the creator owns everything — no usage limits, no privacy concerns, no ongoing costs. Just a machine with a GPU and some open-source models.

Current state:
- 314/314 tests passing ✅
- 48 API endpoints, Web UI (React SPA) + CLI + FastAPI backend
- Full Docker support (ComfyUI sidecar included)
- Production-ready pipeline with batch job management
- Apache 2.0 license

Example output: I produced a 3-episode pilot (18 frames + 18 voiceovers) for a zombie-school anime — all on a MacBook Pro with ComfyUI + Piper TTS.

GitHub: https://github.com/chfr19820610-cell/AIComics

Looking for feedback on:
- UX for non-technical creators (the CLI is functional but a visual editor would help)
- Adding English voiceover support (Piper has English models)
- What features would make this useful for your workflow?
```
