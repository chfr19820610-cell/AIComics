## P0 — 阻塞级问题（Blocking）

### P0-1. 前后端端口不一致，前端API全部不可达
**问题**：前端 `api.ts` 硬编码 API 地址为 `http://127.0.0.1:7860`，但实际后端运行在 `:7861`。端口 `7860` 没有任何进程侦听。

**证据**：
```
前端配置: const apiBaseUrl = 'http://127.0.0.1:7860'  (services/api.ts:34)
实际运行: uvicorn web.backend.app:app --host 127.0.0.1 --port 7861
端口状态: 7860 → NOT LISTENING
```
- 所有 Dashboard 指标、版本信息、任务数据都无法加载（页面显示全部为 0）
- 后端 API 直接请求返回 200（Edition）或 401（Dashboard need auth），说明后端正常工作
- Docker Compose 配置为 `7860:7860`，但手动启动时用了 `--port 7861`

**修复方案**：
1. 在 `serve_frontend_spa.py` 中增加 `/api/*` 反向代理到 `localhost:7861`，或者
2. 修改 `api.ts` 中 `apiBaseUrl` 为 `http://127.0.0.1:7861` 并重新构建前端，或者
3. 统一端口：所有服务统一到 7860（改启动参数 `--port 7860`）

**建议**：方案 1（增加反向代理）最稳妥，无需重建前端。

### P0-2. ComfyUI 工作流文件缺失，AI 图片生成管道阻塞
**问题**：ComfyUI 服务在 `:8188` 正常运行，但 `image_workflow_live_smoke.json` 和 `video_workflow_live_smoke.json` 文件不存在。live_smoke_report.json 确认失败。

**证据**：
```
smoke 状态: failed (v1) / partial (v2)
缺失文件: 
  - local_providers/comfyui/workflows/image_workflow_live_smoke.json ❌
  - local_providers/comfyui/workflows/video_workflow_live_smoke.json ❌
模型根目录: local_providers/comfyui/models/ ❌ 不存在
model_requirements.json 中未注册任何模型需求
```
- Smoke test v2 虽然标记 `passed`，但实际 image provider 的 `preflight_ready: false`，只是靠 `stopped_by_failure_guard` 机制跳过
- 真正的 ComfyUI API workflow JSON 从未被创建过

**修复方案**：
1. 在 ComfyUI 中手动构建 API 格式的 workflow（含 CheckpointLoader + KSampler + VAE Decode + SaveImage），导出为 JSON
2. 复制到 `local_providers/comfyui/workflows/image_workflow_live_smoke.json`
3. 在 `model_requirements.json` 中注册所需 SDXL 模型
4. 下载模型权重到 `local_providers/comfyui/models/` 下

|------|------|--------|

**修复方案**：
- 在连接 API 失败时，显示有意义的错误提示而不是全 0
- 增加 Loading 态骨架屏
- 增加重试按钮和"检查 API 后端"的引导提示

### P0-4. 后端鉴权已启用但前端显示"登录关闭"，用户无法登录
**问题**：后端 Edition 配置 `auth_enabled: true`，Dashboard API 调用返回 `401 Authentication required`，但前端顶部导航显示 `登录关闭`（静态标签），与后端状态矛盾。

**证据**：
```
后端: edition.yaml 中 auth_enabled: true
前端显示: "登录关闭" (static text, not a button)
登陆页: /login 存在但无法通过前端直接到达（无导航入口）
API 请求: http://127.0.0.1:7861/api/dashboard → 401
```
- 用户无法知道需要登录
- 即使知道，也没有入口引导到 `/login`
- 登录页本身存在，但只有从 URL 直接访问才能到达

**修复方案**：
1. 同步 frontend header "登录"状态与后端 auth 配置
2. 在导航栏添加可点击的登录入口
3. 或在开发模式下禁用 auth（修改 edition.yaml）

### P0-5. 启动成本极高——用户需要安装整个 AI 栈
**问题**：从零到产出第一条视频需要大量复杂的环境配置步骤，单个用户几乎不可能完成。

**启动前置要求**（约 12 步）：
```
1. Python 3.12+
2. 创建 venv + pip install -r requirements-lock.txt
3. 安装 ComfyUI（独立仓库）
4. 下载 SDXL 模型权重（~7GB）
5. 下载 Piper TTS 中文模型
6. 配置 ComfyUI workflow JSON（P0-2）
7. 安装 FFmpeg
8. 配置 JWT secret 环境变量
9. 启动 ComfyUI（port 8188）
10. 启动后端 API（port 7861）
11. 启动前端 SPA（port 8000）
12. 创建项目并开始创作
```
- 仅环境搭建就需要 2-6 小时
- 模型下载总量 10GB+
- 无法一键启动

**修复方案**：
1. 编写一键启动脚本 `./start.sh`（检测依赖→启动服务→打开浏览器）
2. 提供 Docker Compose 一键部署方案
3. 为常见模型提供下载脚本（`./download-models.sh`）
4. 将 `serve_frontend_spa.py` 与 `uvicorn` 合并到同一个入口

### P0-6. Gumroad 产品定价为 $0（Pay What You Want），无封面，0 销量
**问题**：Gumroad 上架页设置为了 "Pay What You Want" 模式，默认价格为 $0，且没有产品展示图。

**证据**：
```
价格: $0.00+ (Name a fair price) → 任何人都可以免费获取
默认: $ 输入框为空 / 0
封面图: 使用 Gumroad 默认图片 (assets.gumroad.com/images/opengraph_image.png)

- 无封面图——Gumroad 的商品列表/搜索引擎中不可见
- 英文描述——目标中文用户看不懂/不信任
- 卖的是代码工具包，不是成品视频——买家无法直观看到价值
- 无用户评价/成功案例

3. **描述**：改为中文，增加"已产出 5 集完整漫剧"案例
4. **试看**：嵌入 YouTube/B站 试看视频链接
5. **凭证**：添加 E01-E05 成品的截图轮播
6. **分销**：考虑在知识星球/小报童同步上架

## P1 — 严重级问题（Critical）

- 5 个指标卡：任务成功率/剧集就绪率/手工导入率/重试率/服务商预演率——全部无值

2. 增加"风险提示"区域，展示 2 个已知风险项

### P1-2. Batches 页面功能完整但数据空洞
**问题**：Batches 页面有完整的功能组件（重试操作/导出/归档），但没有任何数据驱动，用户面对空表单不知道要干什么。

- 出现"剧集筛选"和"服务商筛选"的输入框，但没有候选数据

**修复方案**：
1. 用户首次访问时，若无数据，显示"开始首个批次"的引导流
2. 增加"快速开始"卡片引导用户创建项目

### P1-3. 无用户反馈循环——用户不知道系统在做什么
**问题**：所有页面都是静态展示，无实时反馈机制、无进度通知、无操作确认。

**缺失的反馈循环**：
- ❌ 无 toast/消息通知系统（API 返回 401 但用户看不到）
- ❌ 无进度条或加载状态（页面展示全 0 → 用户困惑）
- ❌ 无操作成功/失败反馈（提交后无确认）
- ❌ 无错误边界捕获（JS 错误被吞掉）
- ❌ 无 polling 或 websocket 实时更新

**修复方案**：
1. 接入 antd `message` 全局通知
2. 增加 API 调用拦截器，统一处理错误提示
3. 长时间操作增加 loading 骨架屏
4. 任务状态变化时提供桌面通知（可选）

## P2 — 优化级问题（Improvement）

### P2-1. Documentation/README 与实际代码脱节
**问题**：README 描述了完整的 CLI 工作流，但实际代码路径和命令可能已发生变化。

**例子**：
- `config/web.production.example.yaml` 中 API 端口写的是 7860，实际运行是 7861
- `plan.md` 中引用 Windows 路径 `G:\AIComics\10_System\` 而非 macOS 路径

**修复方案**：全面校对文档路径和端口。

---

### P2-2. 导航栏当前项状态高亮不一致
**问题**：部分页面中当前菜单项没有高亮（如 `/` 总控板未激活，但 `/login` 的导航栏中"总览"项激活）。

**证据**：从 `/login` 导航时，侧边栏"总览"高亮；从 `/` 时，无任何菜单高亮。

**修复方案**：修复 UmiJS route 匹配配置，确保当前路由正确高亮。

---

### P2-3. 缺少移动端/小屏适配
**问题**：当前 UI 使用 ant-design-pro 的 ProLayout 固定侧边栏模式，没有针对窄屏做响应式处理。

**修复方案**：增加侧边栏收起/展开能力，移动端自动切换为抽屉菜单。

---

### P2-4. 所有页面标题（title）为空
**问题**：`document.title` 返回 `"AI漫剧创作者控制台"`，所有子页面（Project/Episodes/Jobs/etc.）在浏览器标签页中显示相同标题，无法区分。

**修复方案**：使用 `document.title` 或 `useEffect` 设置每个路由的独立标题。

---

### P2-5. 用户从零到第一个视频的步骤数过长（约 25 步）
**问题**：即使修复了所有 P0/P1 问题，用户从打开系统到产出第一条漫剧视频仍然需要大量步骤：

```
1. 打开浏览器 → 2. 登录 → 3. 创建项目 → 4. 填写题材/风格/角色
→ 5. 规划剧集数 → 6. 填写故事设定 → 7. 创建角色卡
→ 8. 选择风格模板 → 9. 生成剧本 → 10. 审批剧本
→ 11. 拆镜头 → 12. 检查镜头列表 → 13. 提交批次执行
→ 14. ComfyUI 出图（等待）→ 15. Piper TTS 配音（等待）
→ 16. 素材质检 → 17. 预览生成 → 18. 审核预览
→ 19. 正式版渲染 → 20. 发布包打包 → 21. 发布
```
约 15-21 步，其中 4 步需要等待 AI 生成

**修复方案**：
1. 提供"快速创建"模板（预填题材/角色/风格）
2. 合并登录+项目创建流程
3. 提供"一键从故事到视频"的自动化管道
4. 增加"克隆已有项目"功能

## 0 销量核心归因（Root Cause Analysis）

```
产品端
├─ P0-6: 定价 $0 → 传递"不值钱"信号 → 无人付费
├─ P0-6: 无封面图 → 搜索结果中不可见
├─ P0-6: 英文描述 → 中文目标用户不信任
├─ P0-6: 卖代码包而非成品 → 买家无法直接感知价值
└─ P2: 无用户评价/案例 → 无社会证明

分发端
├─ P1-6: Gumroad 无站外推广
├─ 无小红书/B站/抖音引导 → 无自然流量
├─ 无定价策略（免费无意义）
└─ 产品名称过于技术化 ("AI Comic Book Creator - Complete Toolkit")
    而非面向中文用户的"AI漫剧自动生成工具"

产品端（使用体验）
├─ P0: 全空前端 → 潜在买家即使买了也无法立即看到价值
├─ P0: 无法产出视频 → 买了不能用
└─ P0: 启动成本极高 → 买家大概率放弃

结论：0销量的根本原因是 产品定价错误 + 无曝光 + 买了也用不了 的三重死锁
