## 1. 背景与动机

### 1.1 市场背景

AI 漫剧（Animated Comic Drama）是 2026 年确定性爆发的内容赛道：
- 全球 Webtoon 市场 $14B+，AI 漫画工具市场 $15.9B+（CAGR 33.1%）
- 中国 AI 漫剧市场规模约 240 亿元（2026 年），用户规模突破 2.8 亿
- 抖音平台 AIGC 漫剧 2025 年全年播放量增长 **181 倍**
- YouTube 中文 AI 漫剧频道订阅量已达数十万级，出海变现路径已验证

### 1.2 用户痛点

通过行业调研、竞品分析和用户访谈，当前 AI 漫剧创作者的核心痛点是：

> **"同一角色在不同镜头、不同集之间频繁变形"**

具体表现为：
- 跨镜头脸型/发型/服装不一致（最常见）
- 同一角色的肤色、体型、面部特征随机变化
- 同一场景不同角度的环境元素不匹配
- 连续剧集中角色外观漂移，观众出戏

用户调研（2026年7月）显示：**角色不一致是导致 AI 漫剧弃看率的第一原因**，超过 68% 的观众在发现角色变形后会放弃观看。

### 1.3 竞品差距分析

| 竞品 | 角色一致性方案 | 等级 | 是否支持漫剧视频 |
|------|--------------|------|----------------|
| **LlamaGen.AI** | 专有 LoRA 训练系统，跨页保持 | ★★★★★ | ❌（仅静态漫画） |
| **YarnSaga** | 角色自动锁定（定义一次跨页复用） | ★★★★★ | ❌（仅静态漫画） |
| **Dashtoon** | 内置角色系统 | ★★★★ | ❌（仅 Webtoon） |
| **Anifusion** | AnimagineXL + LoRA | ★★★★ | ❌（仅静态漫画） |
| **KomikoAI** | 内置系统 + 多图生成 | ★★★★ | ✅（Webtoon+动画混合） |
| **可灵 Kling 3.0** | 参考图系统 | ★★★★ | ✅（通用视频） |
| **海螺 AI 2.3** | 首尾帧控制 | ★★★★ | ✅（通用视频） |
| **AIComics** | **❌ 无任何角色一致性系统** | **☆ 待建** | ✅ **原生漫剧视频** |

**关键发现:** AIComics 在所有静态漫画竞品都具备角色一致性系统的市场环境中，**该项能力为 0**。这是 P0 级的缺失功能。

### 1.4 目标

构建 AIComics 内置角色一致性系统，使跨镜头/跨集角色一致率从 ~40%（纯随机）提升至 **90%+**，达到或超过 LlamaGen.AI/YarnSaga 级别的水准。

## 2. 竞品参考

### 2.1 LlamaGen.AI 方案（最佳参考）

- **LoRA 训练管道:** 用户上传 10-20 张角色图 → 自动训练角色专属 LoRA → 所有生成场景引用该 LoRA
- **角色库管理:** 已训练角色集中管理，可跨项目复用
- **角色描述系统:** 文字描述角色特征 + LoRA 双重保障
- **局限:** 训练需要 5-10 分钟，需要 GPU

### 2.2 YarnSaga 方案（最优 UX）

- **角色锁定:** 首次生成时定义角色外貌 → 系统自动锁定 → 后续所有帧自动使用相同外观
- **角色面板:** 可视化管理所有角色（名称、外貌、表情）
- **零训练成本:** 无需训练，Prompt + Reference Image 驱动
- **局限:** 依赖云端模型，表现精度随场景复杂度下降

### 2.3 Dashtoon 方案（最实用）

- **内置角色系统:** 创建时选择角色模板 → 微调外观 → 生成时自动保持一致
- **发布整合:** 角色一旦创建，全平台（发布 App）保持一致
- **局限:** 风格模板化，自由度受限

### 2.4 Midjourney --cref 方案（最轻量）

- **参考图系统:** 上传角色参考图（--cref） → 新生成时自动参考
- **零训练成本:** 不需要训练，即时生效
- **局限性:** 每张新图都需要引用；跨图一致性约 70-80%

### 3.2 核心功能模块

#### Module 1: 角色资产库 (Character Asset DB)

用户创建和管理角色资产的核心存储：

- **角色定义:**
  - 角色名称（必填）
  - 角色外貌描述（LLM 辅助生成的结构化描述：发型/脸型/肤色/体型/服装风格）
  - 角色性格/声音特征（可选，未来用于 TTS 角色化）
  - 标签系统（性别/年龄/剧本/标签分类）

- **参考图管理:**
  - 上传 3-20 张角色参考图（不同角度/表情）
  - 自动裁剪/预处理
  - 图片质量评分
  - IP-Adapter 特征提取并缓存

- **LoRA 资产管理:**
  - 已训练角色 LoRA（safetensors 格式）
  - 训练参数记录（步数/学习率/数据集）
  - LoRA 效果预览（同一 Prompt 下有/无 LoRA 对比）

- **跨项目复用:**
  - 角色资产可跨项目/跨集使用
  - 版本管理（支持多次迭代优化角色外观）

#### Module 2: LoRA 训练管道 (LoRA Trainer)

自动化角色 LoRA 训练服务：

- **训练输入:** 用户上传的 5-20 张角色图
- **预处理:** 自动裁切面部/全身、去背景、标签化
- **训练参数:** 智能默认值（基于图像数量和质量自动计算）
- **训练引擎:** 同时支持两种后端:
  - **轻量级** (内置): 基于 Diffusers + PEFT，纯 Python，无需外部依赖
  - **专业级** (可选): 调用 Kohya_ss / sd-scripts，更高训练质量
- **训练状态:** 实时进度条 + 日志输出
- **训练时间参考:** 5-10 张图约 3-5 分钟（RTX 4090）

**默认配置建议:**
```
模型基底: SDXL / FLUX.1-dev
重复次数: 10
训练步数: 1000-1500
学习率: 1e-4
分辨率: 1024x1024
网络维度: 64 (SDXL) / 32 (FLUX)
```

#### Module 3: 参考图像匹配器 (Reference Image Matcher)

基于 IP-Adapter 的零训练一致性方案（适用于光照训练的场景）：

- **功能:** 上传 1-3 张角色参考图 → 自动提取特征 → 注入 ComfyUI Workflow
- **优势:** 无需训练，即时生效
- **局限:** 一致性约 75-85%，低于 LoRA 方案（90%+）
- **使用场景:** 快速预览、低预算制作、临时角色

#### Module 4: ComfyKit 集成层 (Consistency Engine)

核心编排层，确保所有角色一致性功能无缝集成到现有管道：

- **Prompt 增强:** 自动在生成 Prompt 前插入角色描述前缀
- **LoRA 自动注入:** 根据角色匹配，自动加载对应的 LoRA 到 ComfyUI Workflow
- **参考图自动注入:** 在 Image-to-Image/X-to-Video 管线中自动插入参考图节点
- **跨镜头保证:** 同一分镜表中的多帧共享相同角色配置
- **跨集保证:** 同一项目多集之间自动复用角色资产

#### Module 5: 角色资产管理 UI

前端可视化角色管理界面：

- **角色列表页:** 卡片式展示所有已创建角色，支持搜索/筛选
- **角色详情页:** 角色信息编辑 + 参考图管理 + LoRA 状态
- **角色创建向导:** 分步引导（输入描述 → 上传图片 → 训练 LoRA → 预览效果）
- **效果对比工具:** 同一角色同一 Prompt，有/无 LoRA 的生成结果并排对比
- **批量角色管理:** 一键导入/导出角色资产包

### 3.3 与现有管道的集成

#### Standard Pipeline 集成点

```
用户选择或创建角色 ──→ Script 中包含角色名
                           │
                    ┌──────┴──────┐
                    │ 角色解析    │ ← 从 Script 中提取角色引用
                    └──────┬──────┘
                           │
                    ┌──────┴──────┐
                    │ 角色匹配    │ ← 从 Asset DB 匹配角色定义
                    └──────┬──────┘
                           │
                    ┌──────┴──────┐
                    │ Prompt 增强 │ ← 注入角色描述 + LoRA + 参考图
                    └──────┬──────┘
                           │
                    ┌──────┴──────┐
                    │ ComfyUI     │ ← Workflow 自动加载角色配置
                    │ Workflow    │
                    └──────┬──────┘
                           │
                    ┌──────┴──────┐
                    │ 视频输出    │ ← 角色一致的漫剧视频
                    └─────────────┘
```

#### 与 Novel-to-Pipeline 的关系（方向 3 依赖）

角色一致性系统是小说→漫剧管道的前置依赖：
```
小说原文 → LLM 角色提取 → 匹配已有角色资产 / 自动创建新角色
                                        ↓
                        角色一致性系统（本 PRD 范围）
                                        ↓
                        逐帧生成（角色保持一致）
```

## 6. 技术方案建议

### 6.1 技术选型

| 组件 | 推荐方案 | 备选方案 | 理由 |
|------|---------|---------|------|
| LoRA 训练引擎 | Diffusers + PEFT（内置） | Kohya_ss（外部调用） | Diffusers 零外部依赖、跨平台、易集成 |
| 参考图匹配 | IP-Adapter（ComfyUI 节点） | InstantID / PuLID | 生态兼容性最好，ComfyUI 原生支持 |
| 角色数据库 | SQLite（文件级） | JSON 文件存储 | SQLite 支持关系查询、易于扩展 |
| 特征缓存 | ONNX Runtime（CPU） | torch 推理 | CPU 可运行，降低 GPU 依赖 |
| Prompt 增强 | LLM（Ollama 本地） | 模板替换 | 已有 Ollama 集成，复用现有架构 |
| 一致性评分 | CLIP 特征相似度 | 人工判定 | 自动化验收指标 |

### 6.2 数据模型

```sql
-- 角色定义表
CREATE TABLE characters (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,           -- LLM 生成的结构化外貌描述
    gender TEXT,                -- 男/女/其他
    age_group TEXT,             -- 少年/青年/中年/老年
    tags TEXT,                  -- 逗号分隔标签
    thumbnail_path TEXT,        -- 缩略图路径
    created_at DATETIME,
    updated_at DATETIME,
    project_id TEXT,            -- 所属项目（NULL 表示全局）
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 参考图表
CREATE TABLE reference_images (
    id TEXT PRIMARY KEY,
    character_id TEXT NOT NULL,
    file_path TEXT NOT NULL,     -- 图像文件路径
    angle TEXT,                  -- 正面/侧面/背面/全身
    expression TEXT,             -- 表情描述
    quality_score REAL,          -- 0-1 质量评分
    feature_vector BLOB,         -- IP-Adapter 特征向量缓存
    created_at DATETIME,
    FOREIGN KEY (character_id) REFERENCES characters(id)
);

-- LoRA 模型表
CREATE TABLE lora_models (
    id TEXT PRIMARY KEY,
    character_id TEXT NOT NULL,   -- 对应角色
    file_path TEXT NOT NULL,      -- safetensors 路径
    base_model TEXT,              -- SDXL / FLUX.1-dev
    training_params TEXT,         -- JSON 训练参数
    steps INTEGER,
    learning_rate REAL,
    status TEXT,                  -- training/completed/failed
    consistency_score REAL,       -- 0-1 一致性评分
    created_at DATETIME,
    FOREIGN KEY (character_id) REFERENCES characters(id)
);

-- 项目-角色关联表
CREATE TABLE project_characters (
    project_id TEXT NOT NULL,
    character_id TEXT NOT NULL,
    role_tag TEXT,                -- 在项目中使用的标签，如 "[男主]"
    PRIMARY KEY (project_id, character_id)
);
```

### 6.3 核心接口设计

```python
# Character Asset Service
class CharacterService:
    def create_character(name, description, reference_images) -> Character
    def get_character(character_id) -> Character
    def list_characters(project_id=None, tags=None) -> List[Character]
    def update_character(character_id, updates) -> Character

    def train_lora(character_id, base_model="sdxl") -> LoRAModel
    def get_lora_status(character_id) -> str          # training/completed/failed
    def get_best_lora(character_id) -> LoRAModel      # 自动选最优的已训练 LoRA

    def match_character_by_ref(reference_image) -> Character  # 通过参考图匹配已有角色
    def extract_character_description(image_paths) -> str    # AI 生成角色外貌描述

# Consistency Engine
class ConsistencyEngine:
    def enhance_prompt(base_prompt, character_ids) -> str
    def resolve_script_characters(script_text) -> List[CharacterRef]
    def build_workflow_config(project_id, shot_config) -> WorkflowConfig
    def score_consistency(image_paths, character_id) -> float
```

### 6.4 架构演进路线

**Phase 1 (MVP) — 2周:**
- 角色创建 + 参考图上传 + 基本角色管理 UI

- Script 角色名解析（`[角色名]` → 参考图注入）
- SQLite 资产存储
- **交付:** 基本角色 → 基本一致性（75%+）

**Phase 2 (核心) — 再2周:**
- LoRA 训练管道（Diffusers + PEFT）
- LoRA 自动注入 ComfyUI Workflow
- 角色列表 + 详情前端页面
- 效果对比工具
- **交付:** LoRA 训练 → 强一致性（90%+）

**Phase 3 (增强) — 再2周:**
- 跨项目角色复用
- 批量角色管理（导入/导出）
- 角色问题诊断（一致性评分）
- 角色创建向导（AI 辅助描述生成）
- 训练进度实时反馈
- **交付:** 完整的角色资产生命周期管理
